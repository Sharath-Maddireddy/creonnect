# Infrastructure & Deployment Documentation

This document covers infrastructure setup, environment configuration, deployment workflows, and operational considerations.

## Overview

Creonnect is a **full-stack application** with:
- **Backend:** FastAPI Python application (async)
- **Frontend:** Vite SPA (static)
- **Data Stores:** PostgreSQL (primary), Redis (caching, jobs), pgvector (embeddings)
- **External APIs:** Instagram Graph API, OpenAI (LLM/embeddings), Gemini (vision)

---

## Local Development Setup

### Prerequisites

- Python 3.10+ (for backend)
- Node.js 16+ (for frontend)
- PostgreSQL 13+ (with vector extension)
- Redis 6+ (for caching and RQ)
- Git

### 1. Clone Repository

```bash
git clone https://github.com/yourorg/creonnect.git
cd creonnect
```

### 2. Backend Setup

```bash
# Create and activate virtualenv
python -m venv .venv
source .venv/bin/activate  # On Windows: .\.venv\Scripts\Activate.ps1

# Install dependencies
pip install -r requirements.txt

# Create .env file (copy from .env.example)
cp .env.example .env

# Edit .env with local values:
# - DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/creonnect
# - REDIS_URL=redis://localhost:6379/0
# - OPENAI_API_KEY=sk-...
# - INSTAGRAM_APP_ID=...
# - INSTAGRAM_APP_SECRET=...
# - INSTAGRAM_REDIRECT_URI=http://localhost:8000/api/auth/instagram/callback
```

### 3. Database Setup

```bash
# Start PostgreSQL (if running locally)
# macOS with Homebrew:
brew services start postgresql

# Linux:
sudo service postgresql start

# Windows: Use PostgreSQL installer or Docker

# Create database
createdb creonnect

# Run migrations (if using Alembic or similar)
# alembic upgrade head

# Tables are auto-created by init_db() on app startup
```

### 4. Redis Setup

```bash
# Start Redis (if running locally)
# macOS:
brew services start redis

# Linux:
sudo service redis-server start

# Docker:
docker run -d -p 6379:6379 redis:7-alpine
```

### 5. Backend Development Server

```bash
cd backend
uvicorn main:app --reload --factory
```

Server runs on `http://localhost:8000`. FastAPI docs at `http://localhost:8000/docs`.

### 6. Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Dev server runs on `http://localhost:5173`.

### 7. Access the App

- Frontend: `http://localhost:5173`
- Backend API: `http://localhost:8000`
- API Docs: `http://localhost:8000/docs`

## Environment Configuration

### Core Variables

| Variable | Purpose | Required | Example |
|----------|---------|----------|---------|
| `DATABASE_URL` | PostgreSQL async connection | Yes | `postgresql+asyncpg://user:pass@host:5432/db` |
| `REDIS_URL` | Redis connection | Yes | `redis://localhost:6379/0` |
| `OPENAI_API_KEY` | OpenAI LLM and embeddings | Yes | `sk-...` |
| `GEMINI_API_KEY` | Google Gemini vision (optional) | No | `AIzaSy...` |
| `INSTAGRAM_APP_ID` | Instagram App ID | Yes | `123456789...` |
| `INSTAGRAM_APP_SECRET` | Instagram App Secret | Yes | `abc123...` |
| `INSTAGRAM_REDIRECT_URI` | OAuth callback URI | Yes | `http://localhost:8000/api/auth/instagram/callback` |
| `CREONNECT_SESSION_SECRET` | Session cookie secret (required in prod) | Yes* | Autogenerated in dev; set explicitly in prod |
| `BRAND_API_KEY` | Protected endpoints API key | No | Any string |
| `ENV` | Environment (dev, staging, prod) | No | `dev` |
| `LLM_MODEL_NAME` | LLM model override | No | `gpt-4o-mini` |
| `CORS_ALLOWED_ORIGINS` | Comma-separated CORS origins | No | `http://localhost:3000,https://example.com` |

### Database Configuration

Pool settings (for network databases):
- `DB_POOL_SIZE` — SQLAlchemy pool size (default: 10)
- `DB_MAX_OVERFLOW` — Max overflow connections (default: 20)
- `DB_POOL_TIMEOUT` — Pool timeout seconds (default: 30)
- `DB_POOL_RECYCLE` — Connection recycle seconds (default: 1800)
- `DB_COMMAND_TIMEOUT` — Command timeout seconds (PostgreSQL+asyncpg, default: 10)

### .env File Example

```bash
# Database
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/creonnect

# Redis
REDIS_URL=redis://localhost:6379/0

# APIs
OPENAI_API_KEY=sk-...
GEMINI_API_KEY=AIzaSy...

# Instagram OAuth
INSTAGRAM_APP_ID=123456789
INSTAGRAM_APP_SECRET=abc123...
INSTAGRAM_REDIRECT_URI=http://localhost:8000/api/auth/instagram/callback

# Session
CREONNECT_SESSION_SECRET=my-secret-key-min-32-chars-long

# Optional
BRAND_API_KEY=my-brand-key
ENV=dev
```

## Docker Deployment

### Docker Compose (Local Development)

Create `docker-compose.yml`:

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: creonnect
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

  backend:
    build: .
    command: uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
    environment:
      DATABASE_URL: postgresql+asyncpg://postgres:postgres@postgres:5432/creonnect
      REDIS_URL: redis://redis:6379/0
      OPENAI_API_KEY: ${OPENAI_API_KEY}
      # ... other env vars
    ports:
      - "8000:8000"
    depends_on:
      - postgres
      - redis
    volumes:
      - .:/app

  frontend:
    build: ./frontend
    ports:
      - "5173:5173"
    environment:
      VITE_API_BASE_URL: http://localhost:8000

volumes:
  postgres_data:
```

Run:
```bash
docker-compose up
```

### Dockerfile (Backend)

Create `Dockerfile`:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Dockerfile (Frontend)

Create `frontend/Dockerfile`:

```dockerfile
FROM node:18-alpine as builder

WORKDIR /app
COPY package*.json .
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

## Cloud Deployment (AWS / Heroku / Vercel)

### Heroku Deployment

1. **Create Heroku app:**
   ```bash
   heroku create creonnect-app
   ```

2. **Add PostgreSQL addon:**
   ```bash
   heroku addons:create heroku-postgresql:standard-0
   ```

3. **Add Redis addon:**
   ```bash
   heroku addons:create heroku-redis:premium-0
   ```

4. **Set environment variables:**
   ```bash
   heroku config:set OPENAI_API_KEY=sk-...
   heroku config:set INSTAGRAM_APP_ID=...
   # ... other vars
   ```

5. **Deploy:**
   ```bash
   git push heroku main
   ```

### AWS Deployment (ECS + RDS + ElastiCache)

High-level architecture:
- **ECS Fargate** — Backend container
- **CloudFront** — Frontend CDN
- **RDS PostgreSQL** — Primary database
- **ElastiCache Redis** — Caching
- **ALB** — Load balancer
- **IAM roles** — Security

Detailed steps (use AWS documentation):
1. Build and push image to ECR.
2. Create RDS PostgreSQL instance.
3. Create ElastiCache Redis cluster.
4. Create ECS Fargate task definition.
5. Create ECS service with load balancer.
6. Configure Route 53 DNS.
7. Enable CloudFront for frontend.

### Vercel Deployment (Frontend only)

1. Push frontend to Git.
2. Create Vercel project linked to Git.
3. Set environment variables:
   ```
   VITE_API_BASE_URL=https://api.yourdomain.com
   ```
4. Deploy (auto on push to main).

## Database Migrations

### Using Alembic (SQLAlchemy)

1. **Install Alembic:**
   ```bash
   pip install alembic
   ```

2. **Initialize Alembic:**
   ```bash
   alembic init migrations
   ```

3. **Configure `alembic.ini`:**
   ```ini
   sqlalchemy.url = ...DATABASE_URL...
   ```

4. **Create migration:**
   ```bash
   alembic revision --autogenerate -m "Add user table"
   ```

5. **Apply migration:**
   ```bash
   alembic upgrade head
   ```

### Note

Currently, `init_db()` in `backend/app/infra/database.py` auto-creates tables. For production, consider explicit migrations.

## Monitoring & Observability

### Logging

- All services log via `backend.app.utils.logger.logger`.
- Logs are sent to stdout (container/process logs).
- Integrate with CloudWatch, DataDog, or ELK stack.

### Health Checks

- **Endpoint:** `GET /health`
- **Use:** Configure load balancer health checks to poll `/health`.

### Performance Monitoring

- Use APM (New Relic, DataDog, Sentry) for backend.
- Use Web Vitals libraries for frontend.
- Monitor database query performance.
- Set alerts for high error rates or latency.

### Scaling Considerations

- **Horizontal:** Multiple backend instances behind load balancer.
- **Vertical:** Increase container/instance memory/CPU.
- **Database:** Connection pooling (already configured).
- **Redis:** Cluster mode for high throughput.
- **Caching:** Use CDN for frontend (CloudFront, Vercel, Netlify).

## Backup & Recovery

### Database Backups

```bash
# Local backup (PostgreSQL)
pg_dump creonnect > backup.sql

# Restore
psql creonnect < backup.sql

# AWS RDS: Use automatic snapshots (daily retention)
# Heroku: Use automatic backups or pg:backups
```

### Redis Persistence

- Enable AOF (Append-Only File) or RDB snapshots in redis.conf.
- Cloud providers (ElastiCache, Heroku Redis) handle backups automatically.

## Security Considerations

1. **API Keys:** Store in environment variables, never in code.
2. **CORS:** Restrict to known frontend origins.
3. **HTTPS:** Enforce in production (certificates via Let's Encrypt or AWS ACM).
4. **Rate Limiting:** Implement to prevent abuse (see `backend/app/api/rate_limiter.py`).
5. **CSRF Protection:** Session middleware handles via `SessionMiddleware`.
6. **SQL Injection:** SQLAlchemy ORM prevents (no raw SQL).
7. **Secrets:** Rotate API keys periodically; use secret manager (AWS Secrets Manager, Vault).

## CI/CD Pipeline

### GitHub Actions Example

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: pytest
  
  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v3
      - run: echo "Deploying to production..."
      # Add deployment steps (docker push, heroku deploy, etc.)
```

## Troubleshooting

### Database Connection Errors

```
Error: could not connect to server
```

- Check `DATABASE_URL` format.
- Ensure PostgreSQL is running.
- Check network/firewall rules.

### Redis Connection Errors

```
Error: connection refused
```

- Check `REDIS_URL` format.
- Ensure Redis is running.
- Check firewall rules.

### LLM API Errors

```
Error: OpenAI API key invalid
```

- Verify `OPENAI_API_KEY` is set and valid.
- Check API quota and billing.

### Frontend Build Fails

```
npm ERR! code ENOENT
```

- Run `npm install` to ensure dependencies.
- Check Node version (`node --version`).
- Delete `node_modules` and `package-lock.json`, reinstall.
